## 6. Цепочка ценообразования

```
recommended_works (от Claude)
       │
       ▼
для каждого work_code:
   work.volume   = computeVolume(code, A, P, heightFactor)
   work.price    = base_price × works_coef[region] × finish_factor
   work.total    = volume × price
       │
       ▼
для каждого work → WORK_TO_MATERIALS[code]:
   material.volume = ceil(work.volume / consumption_m2_per_package)   // в упаковках
   material.price  = base_price × materials_coef[region]
   material.total  = volume × price
   if calibration.wear_apply:
       wearK = min(1, renovation_age / service_life_years)
       total = total × (1 − wearK)
       │
       ▼
base = Σ works.total + Σ materials.total
если average_confidence < 0.6:
    base = base × vision_low_confidence_discount
range = [base × (1−σ), base, base × (1+σ)]    // σ из калибровки, по умолчанию 0.15
routed_to_expert = base > stp_threshold_rub
```

Региональные и финиш-коэффициенты — [proto/data/region_coefficients.json](proto/data/region_coefficients.json) и [proto/data/calibration_defaults.json](proto/data/calibration_defaults.json).

## 7. Слабые места в текущем подходе (фиксы — см. §12)

1. **Один материал на одну работу** — нет вариативности по бренду / классу. AI не может сказать «премиум-плитка для премиум-отделки». → §12.1
2. **2 материала висят без работ** (`MAT-PAINT-02`, `MAT-LAM-02`) — попадают в каталог, но никогда не выбираются. → §12.1
3. **`material_predicted` AI не передаётся в калькулятор** — AI определяет тип материала на фото, но цена в смете всегда базовая, не учитывает класс. → §12.3
4. **Один объём `A` на все работы кейса** — если в фото и стена и потолок повреждены, общая площадь приближённо размазывается на всё. AI не разбивает работы пер-фото. → §12.4
5. **Высота потолка влияет только на стены** — `heightFactor` применяется к `PREP/LEVEL/PAINT/WALL` ([calculator.ts:87-93](proto/lib/calculator.ts#L87-L93)). Это спорно для `LEVEL-003/005/007` (это потолочные работы) — сейчас они тоже умножаются на heightFactor, что некорректно. → §12.5

## 8. Критическая оценка каталога материалов (24 кода — недостаточно)

Текущий каталог покрывает базовый сценарий «залив потолка/обоев + локальная замена пола», но **большинство реальных случаев не закрывает**. Ключевые пробелы:

### 8.1 Отсутствуют целые категории

| Категория | Что нужно | Зачем |
|---|---|---|
| **Гидроизоляция** | обмазочная, рулонная, акриловая | санузлы — обязательный слой при перекладке плитки на пол; стяжка после залива |
| **Стяжка / выравниватель** | ЦПС, самовыравнивающаяся смесь, грунт под стяжку | замена пола после длительного залива (бетон под ламинатом сгнил) |
| **Сантехтрубы** | PEX, полипропилен, ПВХ-канализация, фитинги | прорыв стояка / разводки — заменить трубы и потом восстанавливать отделку |
| **Уплотнители / герметики** | силикон санитарный, акриловый, монтажная пена | швы плитки, ванна-стена, уплотнение оконных откосов |
| **Звукоизоляция / утеплители** | минвата, вспененный полиэтилен | при снятии штукатурки на наружной стене — проверить утеплитель |
| **Каркас / профиль для стен** | CD-60+UD-27 для стеновых конструкций | сейчас есть только для потолка (`CEIL-002`) — для перегородок ГКЛ нужен отдельный комплект |
| **Стеновые панели** | ПВХ, МДФ, вагонка | эконом-варианты для коридоров, балконов, санузлов |
| **Влагостойкая отделка** | КГЛ-В (зелёный), плитка-керамогранит, влагостойкая краска | санузлы, кухни — нельзя использовать обычный ГКЛ |
| **Подоконники, отливы** | ПВХ, акриловый камень, дерево | при заливе через окно — заменить подоконник |
| **Антикоррозионные составы** | для металла радиаторов, стояков после ржавчины | если радиатор лопнул и подтекало — обработать соседнюю металлоконструкцию |
| **Вагонка / OSB** | для подсобных помещений | редко, но встречается в загородных квартирах / лоджиях |

### 8.2 Внутри существующих категорий — слишком мало вариантов

| Категория | Сейчас | Нужно |
|---|---|---|
| Краска | 2 кода (один не юзается) | 5–7: эконом ВД, стандарт ВД, премиум, влагостойкая для кухни/ванной, латексная, эмаль для дверей |
| Обои | 1 код (флизелин под покраску) | 4: бумажные эконом, флизелин, винил, текстильные |
| Ламинат | 2 (один не юзается) | 5: ламинат 32 эконом, 33 стандарт, 12 мм комфорт, водостойкий SPC, паркетная доска премиум |
| Плитка стена | 1 код | 3: керамика стандарт, керамогранит крупноформат, мозаика премиум |
| Плитка пол | 1 код | 3: керамика стандарт, керамогранит, премиум |
| Двери | 1 (стандарт МДФ) | 4: эконом, стандарт МДФ, шпонированная комфорт, входная металлическая |
| Шпаклёвка/штукатурка | 2 (универсальные) | 5: ЦПС, гипсовая, влагостойкая для санузла, финишная, ремонтная |

### 8.3 Атрибуты, которых не хватает в схеме

- `tier: "econom" | "standard" | "comfort" | "premium"` — для выбора по уровню отделки клиента
- `applicable_to_surface: ["wall" | "ceiling" | "floor" | "wet"]` — для проверки применимости (нельзя класть напольную плитку на потолок, нельзя клеить обои в санузле)
- `applicable_to_room?: ("bathroom" | "kitchen" | "living")[]` — для water-resistant фильтра в санузлах
- `compatibility_notes` — например, флизелин нельзя клеить на влажную стену сразу после залива (должна высохнуть 7+ дней)

### 8.4 Итоговая оценка

**Сейчас**: 24 материала покрывают ~30–40% реальных случаев залива.
**Цель**: 70–90 кодов с тирами + комнатной/поверхностной аппликабильностью покрывают 80%+.

### 10.2 Алгоритм выбора материала

```text
function pickMaterials(work_code, finish_tier, material_predicted, room_type):
  candidates = WORK_TO_MATERIALS_V2[work_code]
  // 1. Сначала ищем точный матч по material_predicted
  for c in candidates:
    if c.if_material_predicted_in and material_predicted matches → return c.materials
  // 2. Иначе — точный матч по тиру
  for c in candidates:
    if c.tier == finish_tier → return c.materials
  // 3. Fallback: standard
  for c in candidates:
    if c.tier == "standard" → return c.materials
  // 4. Fallback: первая запись
  return candidates[0].materials
```

### 12.1 Один материал на работу → многоуровневый словарь (закрывает п.1, 2)

См. §10. `WORK_TO_MATERIALS_V2` имеет варианты по тирам и `material_predicted`. Неиспользуемые сейчас `MAT-PAINT-02` и `MAT-LAM-02` встают в econom/comfort пути.

### 12.2 `material_predicted` AI не передаётся в калькулятор → расширить схему

```ts
// Сейчас
ClaudeOutput {
  recommended_works: string[];   // только коды работ
  // material_predicted был "пропадает" после parsing
}
// Стало
ClaudeOutput {
  recommended_works: WorkSpec[];
}
interface WorkSpec {
  code: string;
  surface: "ceiling" | "wall" | "floor" | ...;       // нужно для §12.5
  material_predicted?: string;                        // подсказка для §10 dict-а
  area_m2?: number;                                   // нужно для §12.4 (per-photo)
}
```

Задача: обновить `SYSTEM_PROMPT` в [proto/lib/prompts.ts](proto/lib/prompts.ts), чтобы Claude возвращал `recommended_works` как объекты, а не плоский массив строк.

### 12.3 Учёт класса материала при ценообразовании → tier-based selector

В функции `calculate()` ([proto/lib/calculator.ts:177+](proto/lib/calculator.ts#L177)) заменить:

```ts
const matCodes = WORK_TO_MATERIALS[work.code] ?? [];
```

на:

```ts
const matCodes = pickMaterials(
  work.code,
  context.finish_level,
  workSpec.material_predicted,

```

См. алгоритм §10.2.

### 12.4 Один объём `A` на всё → per-surface bundles

Расширение `WorkSpec` (см. §12.2) даёт `area_m2` per-work. Калькулятор использует:

```ts
for (const ws of claudeOutput.recommended_works) {
  const A = ws.area_m2 ?? S_fallback;          // приоритет — per-work, fallback — общая S
  const volume = computeVolume(ws.code, A, P, heightFactor, ws.surface);
  // ...
}
```

Это позволяет AI выдавать «потолок 3 м² + стена 6 м² + пол 8 м²», и каждая работа считается по своей площади.

### 12.5 `heightFactor` к потолочным работам → проверять `surface`

```ts
case "PREP":
case "LEVEL":
case "PAINT":
case "WALL":

```

Должно быть:

```ts
case "PREP":
case "LEVEL":
case "PAINT":
case "WALL":
  return surface === "ceiling" ? A : A * heightFactor;
```

(Передаём `surface` из `WorkSpec` — см. §12.2.) Так `LEVEL-003` (штукатурка потолка), `LEVEL-005` (шпаклёвка потолка), `LEVEL-007` (шлифовка потолка), `PAINT-002` (краска потолка) перестанут зря умножаться на heightFactor.

### 12.6 Бонус: фильтр применимости по комнате

В санузле/кухне обычные обои не клеят — нужно проверить применимость:

```ts
function isApplicable(matCode: string, roomType: string, surface: string): boolean {
  const m = MATERIAL_CATALOG[matCode];
  if (m.applicable_to_surface && !m.applicable_to_surface.includes(surface)) return false;
  if (m.applicable_to_room && !m.applicable_to_room.includes(roomType)) return false;
  return true;
}
```

Если AI выдал `WALL-001` (обои) для санузла — калькулятор отклоняет и берёт fallback (плитка/панели).
